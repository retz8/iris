The Breakdown

  • What it does: Scans valid cut boundaries in a session window, walks backwards accumulating token estimates until a recency budget is met, then picks the earliest valid cut point at or after that position — adjusting for mid-turn splits and compaction markers.
  • Key responsibility: Decides exactly where to truncate session history when oh-my-pi's context window fills, balancing recency against the constraint that cuts must land on clean message or compaction boundaries rather than mid-turn.
  • The clever part: Token accumulation walks backward to find the minimum cut depth, but the cut-point selection then scans forward to the earliest valid boundary at that depth — recovering as much early context as possible while still honoring the recency budget.
