struct pcg_xsl_rr_128_64 { using state_type = absl::uint128; using result_type = uint64_t; inline uint64_t operator()(state_type state) { uint64_t rotate = static_cast<uint64_t>(state >> 122u); state ^= state >> 64; uint64_t s = static_cast<uint64_t>(state); return rotr(s, static_cast<int>(rotate)); } };



Before scrolling: what does this do?

The Breakdown

  • What it does: Implements the XSL-RR output function that converts a 128-bit PCG generator state into a 64-bit random value by XOR-shifting the two halves together and rotating the result by an amount derived from the top 6 state bits.
  • Key responsibility: The output mixer composed into Abseil's absl::BitGen and absl::InsecureBitGen random engines — this struct sits between the raw LCG state update and the final random value returned to callers across Chromium, TensorFlow, and gRPC.
  • The clever part: The rotation amount is extracted from state before the XOR-shift runs, making it statistically independent of the shifted output — this is what distinguishes PCG from naive LCG truncation and prevents the periodic banding a fixed rotation cannot.
